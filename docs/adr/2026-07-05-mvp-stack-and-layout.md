# MVP stack & repo layout

**Date:** 2026-07-05 · **Status:** Accepted · **Context:** bootstrap (Phase 0)

## Decision

Backend: **Python 3.13 + uv**, **FastAPI** + **Pydantic v2**, **SQLAlchemy 2.0
async + Alembic**, **PostgreSQL 18** in Docker. Background work: an
**in-process asyncio job worker** over a `jobs` table claimed with
`FOR UPDATE SKIP LOCKED` — no Redis, no broker. Two hard constraints follow
while this design stands: the api runs **exactly one uvicorn worker process**,
and the worker executes jobs **strictly serially**.

Frontend: **Vite 8 + React 19 + TypeScript (strict)** SPA, served same-origin
by the api container in prod (dev: Vite proxy); **TanStack Query v5 + TanStack
Router v1**; **Tailwind CSS** (major pinned at scaffold); typed API client
generated by **@hey-api/openapi-ts**, drift-checked in CI from Phase 1.

Testing: **pytest** + httpx `ASGITransport`; **Vitest 4** + Testing Library;
**Playwright** CI smoke = boots-and-renders with dummy env and no database. The
golden paste-to-card suite is **local-only** against the compose stack with
config-selectable fake `SourceAdapter`/`ExtractorAdapter` — Gemini is called
server-side in the worker, so browser route-mocking cannot cover it.

Layout: **npm-workspaces monorepo** — root `package.json` is kit tooling +
workspace root; `frontend/` (Vite app) and `backend/` (uv project) land in
Phase 1. The bootstrap PR ships no app code.

## Why

- **Postgres 18** — JSONB for the evolving recipe document plus real relational
  queries later, and native `uuidv7()` (Python 3.13 has no stdlib uuid7): ids
  via `server_default=text("uuidv7()")`. Accepted kit inversion: this local
  Docker DB holds irreplaceable *production* data, not disposable dev state —
  hence hook-guarded volumes, a separate test database for suites
  (docs/TESTING.md), and the Phase 4 backup obligation.
  Not SQLModel: still 0.0.x and pins SQLAlchemy <2.1 — an upgrade trap.
- **No-broker jobs** — one user, minutes-long jobs; a broker buys nothing but
  ops surface. `SKIP LOCKED` claiming is correct at one process and free
  insurance at N. The single-uvicorn-worker and strictly-serial constraints are
  the price of correctness without a broker — they are design invariants, not
  tuning knobs. **Graduation path: TaskIQ** if real concurrency is ever needed.
  Not arq (maintenance-only), not Celery (heavyweight).
- **Vite, not Next.js** — the SPA is one client of the API (the product); SSR
  machinery has zero payoff for a single-user API client. Same-origin serving
  eliminates CORS and any token-in-bundle temptation.
- **npm workspaces** — the kit's tooling (husky, secretlint, battery) assumes
  root-npm; workspaces keep that intact while giving frontend and backend their
  own manifests and letting CI path-filter per toolchain.
- **Exact pins** — several pieces are pre-1.0 or fast-moving (uv, openapi-ts,
  FastAPI); exact versions in lockfiles, upgrades as deliberate PRs.

## Verified

Versions and toolchain health verified against live docs and registries on
2026-07-05; fast-moving items (model ids, pricing, pre-1.0 libs) get
re-verified at implementation time. The stack's working proof is Phase 1's
acceptance gate: `docker compose up` → browser renders API-reported health
through the *generated* typed client, dual-toolchain CI (including the
openapi-ts drift check) green.
